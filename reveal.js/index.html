How to Use Git

For version control and team coordination

Hack Princetion, Fall 2019

By: Andrew Ippoliti

What is Git?

According to Wikipedia

Git is a distributed version-control system for tracking changes in source code during software development. It is designed for coordinating work among programmers, but it can be used to track changes in any set of files. Its goals include speed, data integrity, and support for distributed, non-linear workflows.

Installing Git

https://git-scm.com/downloads

or


						apt-get install git

Configure Git


						git config --global user.name "Your name"
						git config --global user.email "you@gmail.com"

Create a Remote Repository

1. Create it on GitHub (or your provider)

2. Clone it to your local computer: 


						git clone https://github.com/user/repo.git

Check Staging Status


						git status

Record (commit) your Changes


						git add files_to_be_added
						git commit -m 'Clear commit message'

Push your Changes to the Remote Repository


						git push origin master

Create a new Feature (or Experiment or Bug Fix)


						git checkout -b 'name_of_branch'
						# edit files
						git add -u
						git commit -m 'Useful commit message'
						git push origin 'name_of_branch'

Recieve Updates from your Teammates


						git fetch
						git pull

Merge a Branch into Master


						git checkout name_of_branch 
						git merge master

Go Back In Time


						git checkout COMMIT_HASH
						# ... look around ...
						git checkout master

Ignore Files


						nano .gitignore

Determine who did what in a file


					git blame path/to/file
